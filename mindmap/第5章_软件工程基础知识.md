# 第5章 软件工程基础知识 思维导图

```mermaid
mindmap
  root((软件工程基础知识))
    5_1 软件工程 (Software Engineering)
      5_1_1 定义 (Definition)
        Boehm: 构造程序 (Construct Program) 及文档 (Documentation)
        IEEE: 系统化 (Systematic) / 严格约束 (Disciplined) / 可量化 (Quantifiable)
        Fritz Bauer: 经济 (Economical) / 可靠 (Reliable)
        软件危机 (Software Crisis)
          表现: 进度/成本/质量/维护/文档
      5_1_2 软件过程模型 (Software Process Model)
        软件生命周期 (Software Life Cycle)
        瀑布模型 (Waterfall Model)
          特点: 线性 (Linear), 严格串行 (Strictly Sequential), 文档驱动
          缺点: 难以适应变更 (Requirements Change), 风险控制弱
          适用: 需求明确 (Well-defined Requirements)
        原型化模型 (Prototype Model)
          原型开发 (Prototype Development) -> 目标软件开发
          途径: 模拟 (Simulation) / 真正开发 / 现有软件
          类型: 抛弃型 (Throw-away) / 演化型 (Evolutionary)
          适用: 需求模糊 (Fuzzy Requirements)
        螺旋模型 (Spiral Model)
          结合: 瀑布 + 原型 + 风险分析 (Risk Analysis)
          4个象限
            1. 目标设定 (Objective Setting): 需求/约束/计划
            2. 风险分析 (Risk Analysis): 识别/评估/消除
            3. 开发和有效性验证 (Development and Validation)
            4. 评审 (Planning/Review): 评估/计划下一轮
          适用: 大型复杂系统 (Large-scale Systems)
    5_1_3 敏捷模型 (Agile Modeling)
      核心思想
        适应性 (Adaptive) vs 预设性 (Predictive)
        面向人 (People-oriented) vs 面向过程 (Process-oriented)
        迭代增量 (Iterative & Incremental)
      四大价值观 (XP Values)
        交流 (Communication)
        朴素 (Simplicity)
        反馈 (Feedback)
        勇气 (Courage)
      主要方法
        XP (极限编程 - Extreme Programming)
          轻量级 (Lightweight), 螺旋式
        水晶系列 (Crystal Methodologies)
          机动性 (Tactile), 角色/过程模式
        Scrum
          制品: 产品Backlog (Product Backlog), Sprint Backlog
          过程: Sprint (迭代), 每日站会 (Daily Scrum)
        FDD (特征驱动开发 - Feature Driven Development)
          5个核心过程, 首席架构设计师 (Chief Architect)
    5_1_4 统一过程 (RUP - Rational Unified Process)
      特点
        用例驱动 (Use Case Driven)
        以体系结构为中心 (Architecture Centric)
        迭代和增量 (Iterative & Incremental)
      生命周期 (Lifecycle Phases)
        初始 (Inception)
        细化 (Elaboration)
        构造 (Construction)
        移交 (Transition)
      9个核心工作流 (Core Workflows)
        工程: 业务建模 (Business Modeling) / 需求 (Requirements) / 分析设计 (Analysis & Design) / 实现 (Implementation) / 测试 (Test) / 部署 (Deployment)
        支持: 配置与变更管理 (Configuration & Change Management) / 项目管理 (Project Management) / 环境 (Environment)
      4+1 视图 (4+1 View Model)
        逻辑视图 (Logical View)
        实现视图 (Implementation View)
        进程视图 (Process View)
        部署视图 (Deployment View)
        用例视图 (Use Case View)
    5_1_5 软件能力成熟度 (Software Capability Maturity)
      CMMI (Capability Maturity Model Integration) 5级
        1. 初始级 (Initial): 混乱
        2. 已管理级 (Managed): 项目级
        3. 已定义级 (Defined): 组织级
        4. 量化管理级 (Quantitatively Managed): 数据驱动
        5. 优化级 (Optimizing): 持续改进
    5_2 需求工程 (Requirement Engineering - RE)
      需求层次 (Requirement Levels)
        业务需求 (Business Requirement)
        用户需求 (User Requirement)
        功能需求 (Functional Requirement)
        非功能需求 (Non-functional Requirement): 性能/质量/约束
      需求开发 (Requirement Development)
        需求获取 (Elicitation)
          方法: 面谈 (Interview) / 专题讨论会 (Workshop) / 问卷 (Questionnaire) / 原型 (Prototyping) / 头脑风暴 (Brainstorming)
        需求分析 (Analysis)
        需求规格说明 (Specification) -> 需求基线 (Requirement Baseline)
        需求验证 (Validation): 正确性/一致性/完备性
      需求管理 (Requirement Management)
        变更控制 (Change Control)
          过程: 问题分析 -> 变更分析 -> 成本计算 -> 决策 -> 实现
          CCB (变更控制委员会 - Change Control Board)
        需求追踪 (Traceability)
          正向追踪 (Forward Traceability)
          逆向追踪 (Backward Traceability)
          需求跟踪矩阵 (Traceability Matrix)
    5_3 系统分析与设计 (System Analysis & Design)
      5_3_1 结构化方法 (SASD)
        SA (结构化分析 - Structured Analysis)
          DFD (数据流图 - Data Flow Diagram): 数据流 (Data Flow) / 加工 (Process) / 数据存储 (Data Store) / 外部项 (External Entity)
          数据字典 (DD - Data Dictionary)
        SD (结构化设计 - Structured Design)
          基本原则: 信息隐藏 (Information Hiding) / 抽象 (Abstraction) / 模块化 (Modularity)
          耦合 (Coupling) [低->高]: 非直接 (No Direct) / 数据 (Data) / 标记 (Stamp) / 控制 (Control) / 通信 (External) / 公共 (Common) / 内容 (Content)
          内聚 (Cohesion) [高->低]: 功能 (Functional) / 顺序 (Sequential) / 通信 (Communicational) / 过程 (Procedural) / 时间 (Temporal) / 逻辑 (Logical) / 偶然 (Coincidental)
          系统结构图 (SC - Structure Chart)
        SP (结构化编程 - Structured Programming): 顺序/选择/循环
        数据库设计: E-R图 (Entity Relationship Diagram)
      5_3_2 面向对象方法 (OO - Object-Oriented)
        OOA (面向对象分析 - Object-Oriented Analysis)
        OOD (面向对象设计 - Object-Oriented Design)
          类 (Class): 实体类 (Entity Class), 控制类 (Control Class), 边界类 (Boundary Class)
        OOP (面向对象编程 - Object-Oriented Programming)
          特性: 封装 (Encapsulation), 继承 (Inheritance), 多态 (Polymorphism)
        数据持久化 (Data Persistence)
          ORM (Object/Relation Mapping)
          Hibernate / iBatis / JDO (Java Data Objects)
    5_4 软件测试 (Software Testing)
      分类
        静态测试 (Static Testing) vs 动态测试 (Dynamic Testing)
        黑盒测试 (Black-box Testing) vs 白盒测试 (White-box Testing) vs 灰盒测试 (Gray-box Testing)
        人工测试 (Manual Testing) vs 自动化测试 (Automatic Testing)
      阶段
        单元测试 (Unit Testing): 模块
        集成测试 (Integration Testing): 组装
        系统测试 (System Testing): 功能/性能/压力
        验收测试 (Acceptance Testing): Alpha测试 / Beta测试
      其他
        AB测试 (A/B Testing)
    5_5 净室软件工程 (Cleanroom Software Engineering - CSE)
      理论基础
        函数理论 (Function Theory)
        抽样理论 (Sampling Theory)
      技术手段
        统计过程控制下的增量式开发 (Incremental Development)
        基于函数的规范与设计 (Box Structure): 黑盒/状态盒/明盒
        正确性验证 (Correctness Verification)
        统计测试 (Statistically Based Testing) / 使用模型 (Usage Model)
    5_6 基于构件的软件工程 (CBSE - Component-Based Software Engineering)
      构件特征: 可组装 (Composable) / 可部署 (Deployable) / 文档化 (Documented) / 独立 (Independent) / 标准化 (Standardized)
      构件模型 (Component Model): 接口 (Interface) / 元数据 / 部署
      COTS (Commercial-Off-The-Shelf)
      组装 (Composition)
        顺序组装 (Sequential composition)
        层次组装 (Hierarchical composition)
        叠加组装 (Additive composition)
      胶水代码 (Glue Code) / 适配器 (Adapter)
    5_7 软件项目管理 (Software Project Management)
      4P: 人员 (People) / 产品 (Product) / 过程 (Process) / 项目 (Project)
      进度管理 (Schedule Management)
        WBS (工作分解结构 - Work Breakdown Structure)
        甘特图 (Gantt Chart)
      配置管理 (SCM - Software Configuration Management)
        版本控制 (Version Control)
        变更控制 (Change Control)
      质量管理 (Software Quality Management)
        SQA (软件质量保证 - Software Quality Assurance)
        质量认证: ISO 9000
      风险管理 (Risk Management)
        活动: 辨识 / 分析 / 计划 / 追踪 / 控制
```
