# 第4章 信息安全技术基础知识

* **4.1 信息安全基础知识**
  * **4.1.1 概念 (5要素)**
    * **机密性** (Confidentiality)：不暴露给未授权实体
    * **完整性** (Integrity)：未被篡改，可判别
    * **可用性** (Availability)：授权者可访问，不被阻碍
    * **可控性** (Controllability)：控制信息流向和行为
    * **可审查性** (Auditability)：提供调查依据
  * **范围**
    * **设备安全**：稳定性、可靠性、可用性 (物质基础)
    * **数据安全**：秘密性、完整性、可用性 (防止泄露/篡改/毁坏)
    * **内容安全**：政治健康、符合法律/道德
    * **行为安全**：秘密性、完整性、可控性
  * **4.1.2 信息存储安全**
    * **用户标识与验证**：生物特征 (指纹/语音)、拥有物 (IC卡)
    * **用户存取权限限制**：隔离控制 (物理/逻辑)、限制权限 (分级/ACL)
    * **系统安全监控**：审计、日志、进程监控
    * **病毒防治**：补丁、杀毒、隔离、备份
  * **4.1.3 网络安全**
    * **漏洞**：物理接入、软件特权、不兼容性、策略缺失
    * **威胁**：非授权访问、信息泄露、破坏完整性、拒绝服务 (DoS)、病毒传播
    * **目标**：访问控制、认证、完整性、审计、保密

* **4.2 信息系统安全的作用与意义**
  * 国家安全、社会稳定、经济发展的重要组成部分
  * 威胁升级：从“软打击”(窃密) 到 “硬摧毁”(如震网病毒)

* **4.3 信息安全系统的组成框架**
  * **4.3.1 技术体系**
    * **基础安全**：物理环境、芯片、卡
    * **网络安全**：防火墙、VPN、加密传输、认证
    * **OS 安全**：无漏洞、访问控制、最小特权
    * **DB 安全**：完整性、推理控制、多级保护
    * **终端安全**：加密机
  * **4.3.2 组织机构体系**
    * 机构 (决策/管理/执行)、岗位、人事
  * **4.3.3 管理体系**
    * **原则**：“三分技术，七分管理”
    * 法律管理、制度管理、培训管理

* **4.4 信息加解密技术**
  * **4.4.1 基础**
    * **密码学**：编码学 (设计) + 分析学 (破译)
    * **模型**：明文 (P) -> 加密 (E, K) -> 密文 (C) -> 解密 (D, K) -> P
    * **安全性**：取决于计算资源，追求计算上安全 (而非理论上无条件安全)
  * **4.4.2 对称密钥 (共享密钥)**
    * **DES**：64位块，56位密钥，16轮变换。IBM Lucifer改进，NSA制定。
    * **Triple-DES (3DES)**：3次DES，两个密钥 (K1, K2)，有效长度112位。
    * **IDEA**：128位密钥，64位块，8轮迭代。软硬件实现快，抗分析能力强。
    * **AES (Rijndael)**：128/192/256位密钥。NIST标准，免版税。
  * **4.4.3 非对称密钥 (公钥)**
    * **原理**：公钥加密/私钥解密 (保密通信)，私钥加密/公钥解密 (数字签名)
    * **RSA**：基于大素数分解困难。n=pq, 公钥(e,n), 私钥(d,n)。

* **4.5 密钥管理技术**
  * **4.5.1 对称密钥分配**
    * **控制**：标签、控制矢量 (KDC产生)
    * **分配方式**：人工 (小规模)、KDC (Key Distribution Center, 集中式)
    * **KDC结构**：主密钥 (长期) -> 会话密钥 (临时)
  * **4.5.2 公钥管理**
    * **公开发布**：易被欺骗/伪造
    * **公用目录表**：管理员维护，有瓶颈/被攻击风险
    * **公钥管理机构**：类似KDC，也是瓶颈
    * **公钥证书 (Certificate)**：CA签发，包含用户公钥+身份+签名。离线校验，无瓶颈。
  * **4.5.3 混合分配**
    * 利用 **公钥体制** 分配 **对称会话密钥** (解决公钥慢、对称分配难的问题)

* **4.6 访问控制及数字签名**
  * **4.6.1 访问控制**
    * **3要素**：主体 (Subject)、客体 (Object)、控制策略
    * **内容**：认证、策略实现、审计
    * **实现技术**
      * **访问控制矩阵 (ACM)**：行=主体，列=客体。稀疏，难维护。
      * **访问控制表 (ACLs)**：按列存储 (客体视角的权限列表)。流行，查客体方便，查主体难。
      * **能力表 (Capabilities)**：按行存储 (主体视角的“钥匙串”)。查主体方便，查客体难。
      * **授权关系表**：存储非空元素 (元组)。
  * **4.6.2 数字签名**
    * **条件**：可信、不可伪造、不可重用、不可改变、不可抵赖
    * **实现**
      * **对称密钥**：需可信第三方 (仲裁人)
      * **公钥体制**：发送方私钥签名 -> 接收方公钥验证 (最常用)
      * **摘要签名**：对 Hash(文件) 签名，提高效率 (MD5, SHA)

* **4.7 抗攻击技术**
  * **4.7.1 密钥选择**
    * 增大密钥空间 (长度)、选择强钥、随机性 (伪随机数)
  * **4.7.2 拒绝服务 (DoS/DDoS)**
    * **原理**：耗尽资源 (带宽/连接/CPU)，破坏配置
    * **DDoS结构**：Client -> Handler -> Agent -> Victim
    * **SYN Flooding**：利用TCP三次握手缺陷 (半连接)。防御：注册表调整 (Timeout, MaxHalfOpen)。
    * **ICMP攻击**：Ping of Death。
    * **SNMP攻击**：利用默认弱口令控制设备。
  * **4.7.3 欺骗攻击**
    * **ARP 欺骗**：局域网，伪造IP-MAC映射。防御：静态绑定 (arp -s)。
    * **DNS 欺骗**：伪造域名解析IP。检测：被动监听、交叉检查。
    * **IP 欺骗**：伪造源IP (盲攻击)。防御：过滤内网IP进来的外部包。
  * **4.7.4 端口扫描**
    * 全连接 (TCP Connect)、半打开 (SYN, 隐蔽)、FIN (秘密)、第三方 (代理)
  * **4.7.5 漏洞扫描**
    * 基于网络 (远程扫描)、基于主机 (代理 Agent)

* **4.8 保障体系与评估**
  * **4.8.1 安全等级 (GB 17859)**
    * **1级：用户自主保护** (C1)：隔离用户与数据
    * **2级：系统审计保护** (C2)：更细粒度控制，审计，登录 (Win NT/Unix基本水平)
    * **3级：安全标记保护** (B1)：强制访问控制 (MAC)，安全标记
    * **4级：结构化保护** (B2)：形式化模型，隐蔽通道分析，结构化TCB
    * **5级：访问验证保护** (B3)：访问监控器 (抗篡改、足够小、可分析)
  * **4.8.2 风险评估**
    * **要素**：资产 (价值)、威胁 (可能性)、脆弱性 (严重程度)
    * **关系**：威胁 利用 脆弱性 导致 资产损失 (风险)
    * **风险计算**：Risk = f(Asset, Threat, Vulnerability)
    * **类型**：自评估、他评估 (检查)
