# 第12章 信息系统架构设计理论与实践

- **12.1 概述**
  - **12.1.1 概念**
    - **定义**：软件系统的结构、行为和属性的高级抽象 (元素 + 交互 + 模式 + 约束)
    - **Zachman框架**：6个视角 (规划者/所有者/设计者/建造者/承建者/使用者) x 6个维度 (数据/功能/网络/人员/时间/动机)
  - **12.1.2 发展**
    - **FEAF (联邦企业架构框架)**：业务/运营/技术/变革
    - **TOGAF**：开放组架构框架 (Open Group)

- **12.2 信息系统架构 (ISA)**
  - **12.2.1 架构风格**
    - 数据流、调用/返回、独立构件、虚拟机、仓库 (同第7章)
  - **12.2.2 分类**
    - **物理结构**：
      - **集中式**：资源集中，管理方便但脆弱
      - **分布式**：资源分散，扁平化/网络化 (C/S, B/S)
    - **逻辑结构**：
      - 功能子系统综合体 (横向/纵向/纵横综合)
  - **12.2.4 常用4种架构模型**
    - **(1) 单机应用模式 (Standalone)**
      - 运行在一台物理机上，无网络支持
      - **应用**：早期专业软件 (CAD/PhotoShop)
    - **(2) 客户机/服务器模式 (C/S)**
      - **两层 C/S**：Fat Client (前台界面+业务逻辑) + 后台数据库
      - **三层 C/S**：Client + App Server + DB Server
      - **B/S (Browser/Server)**：瘦客户端，HTTP协议，零维护
      - **多层 C/S**：引入中间件 (负载均衡/安全隔离/连接复用)
    - **(3) MVC 结构**
      - **M (Model)**：数据/业务逻辑 (EJB/Hibernate)
      - **V (View)**：展示 (JSP/HTML)
      - **C (Controller)**：控制流/分发 (Servlet/Struts/SpringMVC)
    - **(4) 面向服务架构 (SOA)**
      - **SOA**：粗粒度服务，松耦合，标准接口
      - **Web Service**：SOAP, WSDL, UDDI
      - **ESB (企业服务总线)**：消息/转换/路由/连接器 (支撑SOA)
  - **12.2.5 总体框架 (金字塔)**
    - **第一层 (战略系统)**：高层决策支持 (DSS)，长期/短期规划
    - **第二层 (业务系统)**：业务流程重组 (BPR)，战术管理
    - **第二层 (应用系统)**：MIS/ERP/CRM等，功能实现
    - **第三层 (信息基础设施)**：技术 (硬/软/网)、信息资源 (数据)、管理 (制度/人)

- **12.3 架构设计方法**
  - **12.3.1 ADM (TOGAF)**
    - **4大架构领域**：
      - **业务架构 (Business)**：战略/治理/流程
      - **数据架构 (Data)**：逻辑/物理数据资产
      - **应用架构 (Application)**：应用系统部署/交互
      - **技术架构 (Technology)**：软硬件/网络基础设施
    - **ADM 周期 (详见图12-9)**：
      - **预备**：框架/原则定义
      - **A (愿景)**：范围/利益相关者/KPI
      - **B (业务)** -> **C (信息系统:数据+应用)** -> **D (技术)**：基线 -> 目标 -> 差距分析
      - **E (机会/方案)** -> **F (迁移规划)**：实施策略/路线图
      - **G (实施治理)** -> **H (变更管理)**：监控/合规/演进
    - **核心**：需求管理 (动态过程，贯穿始终)
  - **12.3.2 信息化总体架构**
    - **信息化特征**：易用性、健壮性、平台化/灵活性、安全性、整合性、移动性
    - **架构模式**：
      - **数据导向**：关注数据模型/质量/生命周期 (BI/MDM)
      - **流程导向**：关注端到端流程/服务编排 (SOA) -> 需结合MDM
    - **建设生命周期**：
      - 规划 (可行性) -> 分析 (逻辑模型/做什么) -> 设计 (物理模型/怎么做) -> 实施 (编码/测试) -> 运维 (评价/修改)
    - **规划方法论**：
      - **CSF (关键成功因素法)**：抓主要矛盾，识别关键信息集合 (适合管理目标)
      - **SST (战略目标集转化法)**：战略目标 -> 信息系统目标 (全面/结构化)
      - **BSP (企业系统规划法)**：U/C矩阵，自上而下识别，自下而上设计

- **12.4 案例分析**
  - **价值驱动体系结构**：价值模型 (期望值/反作用力/变革催化剂)
  - **Web Services在HL7应用**：医疗标准，RIM模型，适配器开发
  - **以服务为中心的企业整合**：航空公司案例 (Ramp Control) -> 服务建模/ESB集成
