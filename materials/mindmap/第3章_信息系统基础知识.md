# 第3章 信息系统基础知识

- **3.1 信息系统概述**
  - **3.1.1 定义**
    - **组成**：硬件、网络、软件、资源、用户、规章制度
    - **目的**：处理信息流，支持决策和控制
    - **5大功能**：输入、存储、处理 (OLAP/DM)、输出、控制
    - **性质**：以计算机为基础的人机交互系统
  - **3.1.2 发展 (诺兰模型)**
    - **初始阶段**：办公设备，个别人使用，财务部门
    - **传播阶段**：需求增加，数据处理发展，出现数据冗余/不一致
    - **控制阶段**：IT投资控制，解决数据共享，部门内网络化 (信息孤岛)
    - **集成阶段**：统一规划，建立基础DB，资源集成共享
    - **数据管理阶段**：统一DB平台，全系统资源整合，管理高效
    - **成熟阶段**：满足各层需求，支持决策，IT与管理融合
  - **3.1.3 分类**
    - **TPS** (业务/数据处理)：局部事务，最低层
    - **MIS** (管理信息系统)：全局性，辅助管理，人机系统
    - **DSS** (决策支持)：半/非结构化问题，交互式
    - **ES** (专家系统)：智能程序，专家知识+推理，模拟决策
    - **OAS** (办公自动化)：综合性，事务处理，提高效率
    - **综合性 (ERP/WMS/MES/PDM)**：融合，互补
  - **3.1.4 生命周期**
    - **产生**：概念产生，需求分析
    - **开发**：总体规划 -> 系统分析 (逻辑模型) -> 系统设计 (实施方案) -> 系统实施 (编码) -> 系统验收
    - **运行**：试运行，维护 (排错/适应/完善/预防)
    - **消亡**：更新改造，报废重建
  - **3.1.5 建设原则**
    - **高层管理人员介入** (CIO出现)
    - **用户参与开发** (全过程、深度参与)
    - **自顶向下规划** (保证信息一致性)
    - **工程化** (引入软件工程/系统工程)
    - 其他：创新性、整体性、发展性、经济性
  - **3.1.6 开发方法**
    - **结构化方法**：生命周期划分，自上而下，文档规范，适合成熟系统
    - **原型法**：快速建立模型，反复修改，用户需求不明确时有效
    - **面向对象方法 (OO)**：对象、类、封装、继承、多态，构建问题域模型
    - **面向服务 (SOA)**：构件、接口、解耦，跨平台/互操作

- **3.2 业务处理系统 (TPS)**
  - **3.2.1 概念**
    - 最初级形式，服务于作业层 (最低层)
    - 针对具体事务 (记账、库存等)
    - 高度结构化，可预先确定
  - **3.2.2 功能**
    - **数据输入**：人工/自动 (POS, OCR, ATM, 扫描)
    - **数据处理**
      - **批处理** (Batch)：定期处理，经济，适合薪资/报表
      - **联机事务处理** (OLTP)：实时，即时更新，高成本/容错
    - **数据库维护**：检索、修改、存入、删除
    - **文件/报表生成**：行动文件、信息文件、周转文件
    - **查询处理**：有条件检索
  - **3.2.3 特点**
    - 信息产生器 (为MIS/DSS提供基础)
    - 企业边界桥梁
    - 结构化程度高

- **3.3 管理信息系统 (MIS)**
  - **3.3.1 概念**
    - TPS基础上发展，高度集成化
    - **结构**
      - 开环 vs 闭环 (反馈)
      - 金字塔结构 (战略/管控/运控/业务)
  - **3.3.2 功能**
    - 功能-过程结构：如销售、生产、后勤等子系统间的联系
  - **3.3.3 组成 (功能/层次矩阵)**
    - **子系统**：销售市场、生产、后勤、人事、财务会计、信息处理、高层管理
    - **软件结构**：公用程序、专用文件、模型库、DBMS等

- **3.4 决策支持系统 (DSS)**
  - **3.4.1 概念**
    - **定义**：支持半结构化/非结构化决策，交互式，提高有效性
    - **基本模式**：管理者 + 真实系统 + DSS (对话/DB/模型库)
    - **结构**：两库结构 (DB+模型库)，或基于知识的结构
  - **3.4.2 功能**
    - 数据整理/提供，外部信息收集，反馈信息，模型/方法管理，交互/图形输出，分布使用
  - **3.4.3 特点**
    - 面向决策者
    - 支持半结构化问题
    - 辅助而非代替
    - 动态性
    - 交互式
  - **3.4.4 组成**
    - **数据重组和确认** (数据仓库)
    - **数据字典**
    - **数据挖掘和智能体** (联合/定序/分类/聚类)
    - **模型建立** (枚举/算法/启发式/模拟)

- **3.5 专家系统 (ES)**
  - **3.5.1 概念**
    - **公式**：知识 + 推理 = 系统
    - **要素**：综合数据库 + 知识库 + 推理机
    - **AI 分支**：ES、机器人、视觉、NLP、学习系统、神经网络
  - **3.5.2 特点**
    - 超越时间限制
    - 操作成本低
    - 易传递复制
    - 处理手段一致
    - 善于克服难题
    - 适用特定领域
  - **3.5.3 组成**
    - **知识库** (静态)：事实性 + 启发性
    - **综合数据库** (动态)：初始状态、中间结果
    - **推理机**：正向 (数据驱动)、反向 (目标驱动)、双向
    - **知识获取**：编辑求精、自学习
    - **解释程序**：回答用户提问，验证推理
    - **人机接口**

- **3.6 办公自动化系统 (OAS)**
  - **3.6.1 概念**
    - 信息处理手段自动化
    - **4大支柱**：行为科学(主导)、系统科学、计算机技术、通信技术
  - **3.6.2 功能**
    - **事务处理**：单机 (文字/日程/表)、多机 (会议/邮件/语音)
    - **信息管理** (管理型)：信息流控制
    - **辅助决策** (决策型)：综合分析
  - **3.6.3 组成**
    - 计算机设备 (主机/终端/外设)
    - 办公设备 (电话/传真/复印)
    - 数据通信及网络
    - 软件系统 (系统/专用/支持)

- **3.7 企业资源规划 (ERP)**
  - **3.7.1 概念**
    - **三流集成**：物流、资金流、信息流
    - **发展**：MRP -> MRP II -> ERP (面向供应链, 扩展到全行业)
  - **3.7.2 结构 (主要模块)**
    - **生产预测**：德尔菲、移动平均
    - **销售管理**
    - **生产计划大纲 (PP)**：承上启下
    - **主生产计划 (MPS)**：细化PP，生产什么/多少/何时
    - **物料需求计划 (MRP)**：核心，推导零部件/采购件需求
    - **能力需求计划 (CRP)**：发现瓶颈
    - **车间作业计划 (PAC)**：执行层，JIT
    - **采购与库存**
    - **质量与设备** (TQM)
    - **财务管理**：渗透全过程
    - **扩展**：CRM, SCM, 电子商务
  - **3.7.3 功能**
    - 支持决策 (各个层面)
    - 行业解决方案 (扩展到非制造业)
    - 供应链管理 (全行业/跨行业)

- **3.8 典型信息系统架构模型**
  - **3.8.1 电子政务**
    - **概念**：政府形态改造，公务/政务/商务/事务一体化
    - **互动领域**：G2G, G2B (对企业), G2C (对居民), B2G (企业对政府), C2G (居民对政府)
    - **技术形式 (4阶段)**：起步(发布) -> 单向互动 -> 双向互动 -> 网上事务处理
    - **应用**：面向社会、部门间、内部办公、核心数据、采购、社区
  - **3.8.2 企业信息化**
    - **概念**：利用IT技术，局部到全局，战术到战略
    - **目的**：提高竞争力 (技术/管理/制度创新)
    - **规划融合 (3层面)**：企业战略、业务运作 (关键流程)、管理运作
    - **方法**
      - 业务流程重构 (BPR)
      - 核心业务应用
      - 信息系统建设
      - 主题数据库 (面向核心业务)
      - 资源管理 (ERP, SCM)
      - 人力资本投资 (智力密集型)
